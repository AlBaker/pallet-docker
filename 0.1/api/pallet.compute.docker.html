pallet.compute.docker documentation

The docker provider allows Pallet to use and configure Docker containers.

For configuration we assume any image has an SSH server installed. During configuration the SSH server is run as PID 1. By default, pallet starts containers with the /usr/sbin/sshd -D command, which can be overridden in the :init key of the :image.

The pallet/ubuntu image has openssh-server installed, and the /var/run/sshd directory created. Password is pallet.

Docker doesn't have support for setting any sort of per node metadata, so we have to maintain this in a file on the host node.
